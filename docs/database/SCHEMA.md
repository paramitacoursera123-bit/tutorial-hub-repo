## Collections Detail

### 1. adminUsers Collection

**Purpose:** Store OAuth (Google/GitHub) users who have admin access

**Document ID:** Firebase Auth UID (automatically set)

**Example Document:**
```javascript
{
  uid: "abc123xyz456",              // Firebase Auth UID (document ID)
  email: "admin@example.com",       // Email address
  displayName: "John Doe",          // Display name from OAuth
  photoURL: "https://...",          // Profile picture from OAuth
  role: "admin",                    // Always "admin"
  providerData: [
    {
      provider: "google.com",       // OAuth provider
      uid: "google-uid-123",        // Provider-specific UID
      email: "admin@example.com",
      displayName: "John Doe",
      photoURL: "https://...",
      phoneNumber: null,
      isAnonymous: false,
      metadata: { ... }
    }
  ],
  createdAt: Timestamp,             // Account creation time
  lastLogin: Timestamp              // Last login time
}
```

**Fields:**
| Field | Type | Description |
|-------|------|-------------|
| `uid` | string | Firebase Auth UID (doc ID) |
| `email` | string | User email |
| `displayName` | string | Display name |
| `photoURL` | string | Profile picture URL |
| `role` | string | Always "admin" |
| `providerData` | array | OAuth provider information |
| `createdAt` | timestamp | Creation date |
| `lastLogin` | timestamp | Last login date |

**Rules:**
- ✅ Each OAuth user has one document
- ✅ Document ID = Firebase Auth UID
- ✅ Role always "admin"
- ✅ Created automatically on first login
- ✅ Updated on each login (lastLogin)

**Indexes:** None required (queried by UID only)

### 2. users Collection

**Purpose:** Store email/password users

**Document ID:** User email (unique)

**Example Document:**
```javascript
{
  email: "user@example.com",        // Email (document ID)
  displayName: "Jane Smith",        // Display name
  role: "user",                     // Always "user"
  createdAt: Timestamp,             // Account creation time
  lastLogin: Timestamp              // Last login time
}
```

**Fields:**
| Field | Type | Description |
|-------|------|-------------|
| `email` | string | User email (doc ID) |
| `displayName` | string | Display name |
| `role` | string | Always "user" |
| `createdAt` | timestamp | Creation date |
| `lastLogin` | timestamp | Last login date |

**Rules:**
- ✅ Email is document ID (unique)
- ✅ Role always "user"
- ✅ Created on signup
- ✅ Updated on each login (lastLogin)
- ✅ No password stored (managed by Firebase Auth)

**Indexes:** None required (queried by email only)

### 3. tutorials Collection

**Purpose:** Store tutorial content

**Document ID:** Auto-generated by Firestore

**Example Document:**
```javascript
{
  id: "tutorial-abc123",            // Document ID (auto-generated)
  title: "React Hooks Guide",       // Tutorial title
  description: "Learn React Hooks", // Short description
  content: "# React Hooks\n...",    // Markdown content
  category: "react",                // Category ID (link to categories)
  authorId: "uid-of-author",        // Admin user ID
  createdAt: Timestamp,             // Creation date
  updatedAt: Timestamp,             // Last update date
  published: true,                  // Publish status
  viewCount: 1234,                  // Number of views
  tags: ["react", "hooks", "state"],  // Search tags
  difficulty: "intermediate"        // Difficulty level
}
```

**Fields:**
| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique tutorial ID |
| `title` | string | Tutorial title |
| `description` | string | Short description |
| `content` | string | Markdown content |
| `category` | string | Category ID |
| `authorId` | string | Author's Firebase UID |
| `createdAt` | timestamp | Creation date |
| `updatedAt` | timestamp | Last updated |
| `published` | boolean | Is published |
| `viewCount` | number | View count |
| `tags` | array | Search tags |
| `difficulty` | string | Difficulty level |

**Rules:**
- ✅ Created by admins only
- ✅ Public read access
- ✅ Admin write access
- ✅ Auto-increment viewCount on access

**Indexes Required:**
- Composite: `published` + `createdAt` DESC
- Composite: `category` + `published`
- Single: `tags` (array index)

---

### 4. categories Collection

**Purpose:** Store tutorial categories/topics

**Document ID:** Category slug (e.g., "react", "node-js")

**Example Document:**
```javascript
{
  id: "react",                      // Category ID (document ID)
  name: "React",                    // Display name
  description: "React tutorials",   // Description
  icon: "📚",                       // Category icon/emoji
  color: "#61DAFB",                 // Brand color
  createdAt: Timestamp,             // Creation date
  updatedAt: Timestamp              // Last updated
}
```

**Fields:**
| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Category slug |
| `name` | string | Display name |
| `description` | string | Description |
| `icon` | string | Icon/emoji |
| `color` | string | Brand color |
| `createdAt` | timestamp | Creation date |
| `updatedAt` | timestamp | Last updated |

**Rules:**
- ✅ Public read access
- ✅ Admin write access
- ✅ Few documents (static reference data)

**Indexes:** None required

## Query Examples

### Find admin by email
```javascript
const q = query(
  collection(db, 'adminUsers'),
  where('email', '==', 'user@example.com')
);
```

### Get all tutorials in category
```javascript
const q = query(
  collection(db, 'tutorials'),
  where('category', '==', 'react'),
  where('published', '==', true)
);
```

### Get recent tutorials
```javascript
const q = query(
  collection(db, 'tutorials'),
  where('published', '==', true),
  orderBy('createdAt', 'desc'),
  limit(10)
);
```

### Search tutorials by tags
```javascript
const q = query(
  collection(db, 'tutorials'),
  where('tags', 'array-contains', 'react'),
  where('published', '==', true)
);
```

## CRUD Operations

### Create
```javascript
// New tutorial
await setDoc(doc(db, 'tutorials', tutorialId), {
  title: "...",
  content: "...",
  createdAt: serverTimestamp(),
  updatedAt: serverTimestamp()
});
```

### Read
```javascript
// Get single tutorial
const doc = await getDoc(doc(db, 'tutorials', tutorialId));
const data = doc.data();
```

### Update
```javascript
// Update tutorial
await updateDoc(doc(db, 'tutorials', tutorialId), {
  title: "New Title",
  updatedAt: serverTimestamp()
});
```

### Delete
```javascript
// Delete tutorial
await deleteDoc(doc(db, 'tutorials', tutorialId));
```

## Security Rules

### Rules by Collection

**adminUsers:**
```
- Read: Own document OR any admin
- Write: Own document only
```

**users:**
```
- Read: Own document
- Write: Own document (except role)
```

**tutorials:**
```
- Read: Public (published = true)
- Write: Admin only
```

**categories:**
```
- Read: Public
- Write: Admin only
```
